※클라우드 개념
●클라우드 컴퓨팅: 인터넷을 통해 IT리소스와 애플리케이션을 on-demand(요구만 있으면)로 제공하는 서비스. 사용한 만큼만 청구.


●On-premise환경 : 기업의 서버를 자체적으로 보유한 전산실 서버에 직접 설치해 운영하는 방식
장점:보안
단점:많은 비용과 시간
●클라우드환경 : 네트워크,서버,스토리지,애플리케이션,서비스 등 컴퓨팅 리소스의 공유풀에 대해 편리하고 언제든지 엑세스 할 수 있으며,
최소의 관리 노력 또는 서비스 제공업체 간의 상호 작동에 의해 신속하게 제공되어 이용할 수 있는 모델 중 하나.


●가상화
 : 물리적인 하드웨어 장치를 논리적인 객체로 추상화 하는 것. 
 : 하나의 하드웨어를 여러 개로 동작시키거나 반대로 여러 개의 장치를 묶어 하나의 장치인 것처럼 사용자(개발자)에게 공유자원으로 제공 가능.

○클라우드 배치모델(Deploy Model): 클라우드를 시스템의 설치 장소에 따라 분류한 개념(서비스 운용에 따른 분류)

-퍼블릭 클라우드 : 여러 사용자와 기업이 인터넷을 통해 공유하면서 사용

-프라이빗 클라우드 : 기업이 시스템 자원을 직접 소유하고 자신들의 회사 전용의 클라우드로 사용하는 것

-가상 프라이빗 클라우드 : 다른 사용자와 함께 사용하거나 인터넷을 이용하는 방식이 보안 측면에서 걱정되지만 퍼블릭 클라우드의 장점인 cost-performance를 통한
프라이빗 클라우드를 구축하고 싶을 때 적용하는 기술

-하이브리드 클라우드 : 퍼블릭 클라우드와 프라이빗 클라우드 각각의 장점을 적용한 시스템

○클라우드 서비스 모델
Iaas(Infrastructure as a Service): 서버, 스토리지 같은 시스템 자원을 클라우드로 제공하는 서비스	
ex)네이버클라우드, 아마존EC2, 마이크로소프트 애저, 구글 컴퓨트 엔진
가상화+서버+스토리지+네트워크
↓

Paas(Platform as a Service) :앱을 개발하거나 구현할 때, 관련 인프라를 만들고 유지보수하는 복잡함 없이 애플리케이션을 개발, 실행, 관리 할 수 있게하는 플랫폼 제공
ex)구글,네이버,다음에서 제공하는 공개 API. MS Azure, Google App Engine
가상화+서버+스토리지+네트워크+★운영체제+미들웨어+런타임★
↓

Saas(Software as a Service) : 소프트웨어 및 관련 데이터는 중앙에 호스팅되고 사용자는 웹 브라우저 등의 클라이언트를 통해 접속하는 형태의
소프트웨어 전달 모델
-->인터넷만 연결되면 바로 어플리케이션 사용 가능 ex)구글 DOCS, 구글 스프레드시트, MS Office 365
가상화+서버+스토리지+네트워크+운영체제+미들웨어+런타임+★데이터+애플리케이션★ 

●클라우드컴퓨팅
주문형 셀프서비스 : 사용자 개입 없이 리소스 확보
광범위한 네트워크 액세스 : 어디서나 액세스
리소스 풀링 : 제공업체가 소비자와 리소스 공유
신속한 유연성 : 필요에 따라 빠르게 추가 리소스 확보
측정형 서비스 : 사용한 만큼만 지불

●클라우드 목표
물리적/colocation --> 가상화 --> 서버리스

 - CLI(Command Line Interface), CUI(Character User Interface), TUI(Text User Interface)

+리눅스 계정
- 보안을 위해 로그인시 패스워드 요구 
- 유닉스/리눅스에서는 기본적으로 세 가지 형태의 계정이 있음

1) root - 관리자 계정
: 거의 모든 권한을 행사할 수 있고 시스템 관리 목적의 계정

2) system 계정 - 시스템 운영에 사용되는 계정 
: 로그인 불가

3) 일반 사용자 계정 - 필요에 따라 생성된 계정
: 로그인이 허락되는 계정
: 보안상의 이유로 시스템 사용에 많은 제약을 받는다

+ 리눅스 명령어
- 대소문자를 구분 
- 기본형태 : 명령어 - 옵션 인수 
- 명령어 도움 : 명령어 -- help
 
+PATH
-절대경로: 파일 및 디렉토리 위치를 '/'(루트디렉토리) 기준으로 표시
-상대경로: 파일 및 디렉토리 위치를 현재 디렉토리 기준으로 표시 
(. : 현재 디렉토리 | .. : 상위 디렉토리)
